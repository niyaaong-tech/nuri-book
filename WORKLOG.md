## 배포 프로세스

Claude가 index.html 수정 → GitHub push → Netlify 자동 배포 (30초) → URL 새로고침

### Claude에게 수정 요청 시
수정 내용 채팅으로 전달 → Claude가 index.html 수정 후 push → 30초 대기 → 확인

### 이미지 교체 시
1. 구글 드라이브 폴더("별마름")에 새 파일 업로드
2. 스프레드시트 C열 해당 행의 파일 ID 업데이트
3. HTML 내 `FILE_IDS` 객체 해당 번호 수정 요청 (또는 시트 자동 반영 — 페이지 로드마다 시트 fetch 시도)

### 고해상도 이미지로 교체 시
드라이브의 파일 ID가 바뀌면 스프레드시트 C열 업데이트만 하면 됨.
HTML은 페이지 로드 시 시트에서 최신 ID를 읽어옴. (실패 시 HTML 내장 ID 폴백)

## 현재 구조 (index.html)

### 뷰 모드
| 모드 | 설명 |
|---|---|
| 서적뷰 (기본) | 좌우 펼침 스프레드 16면, 키보드 화살표 / 버튼 이동 |
| 웹툰뷰 | 세로 스크롤, 1페이지 단위 연속 |
| 전환 버튼 | 표지 우상단 토글 버튼 (스크롤 내리면 사라짐) |

### 이미지 로딩
- `FILE_IDS` 객체에 0~26번 파일 ID 하드코딩 (폴백)
- 페이지 로드 시 스프레드시트 CSV fetch 시도 → 성공 시 최신 ID로 갱신
- 이미지 3단 폴백: `uc?export=view` → `thumbnail` → `lh3`

### 텍스트 배치 (웹툰뷰)
| 방식 | 장면 |
|---|---|
| 이미지 오버레이 | 1·3·4·5·6·11·12·14·15·19·22·24·25·26 |
| 이미지 하단 갭 패널 | 2·7·8·9·10·13·16·17·18·20·21 |
| 무자 | 23 (별의 폭발 — 침묵) |

### 서적뷰 스프레드 구성 (16면)
| 펼침 | 구성 |
|---|---|
| 1 | 표지 |
| 2~4 | [1\|2] [3\|4] [5\|6] |
| 5 | [텍스트면\|7] ⚠️검토 |
| 6~8 | [8\|9] [10\|11] [12\|13] |
| 9 | [14\|15] |
| 10 | [어둠면\|16] ⚠️검토 (침묵 스프레드) |
| 11~12 | [17\|18] [19\|20] |
| 13 | [21\|22] |
| 14 | [23 풀블리드] ⚠️검토 (무자) |
| 15 | [24\|25] |
| 16 | [26\|간기] |

⚠️ 검토 배지 3건: 디렉터 판정 후 확정
